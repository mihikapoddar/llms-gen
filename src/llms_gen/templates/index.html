Payload

HTTPS POST with Content-Type: application/json. Sent only after a monitor crawl finishes and the new llms.txt SHA-256 differs from the stored one. The first successful crawl for a monitor does not send a webhook. On this form, a webhook URL is required whenever Monitor this site is checked, so you are not left without change notifications.

Field Meaning
event Always llms_txt.changed
site_id Monitor id in this app
root_url Site root that was crawled
job_id Completed crawl job id
sha256 Digest of the new llms.txt
previous_sha256 Digest from the prior monitor run
content_changed_at ISO 8601 time when the change was recorded
artifact_path URL or path to fetch the file (set public base URL in server config for an absolute link)

Download the generated file with GET /api/artifacts/{job_id} using the same job_id as in the JSON.

Zapier

  • New Zap → Webhooks by Zapier → Catch Hook → paste the hook URL into the webhook field above.
  • Run a monitored crawl so Zapier receives a sample JSON, then map fields to Email, Slack, or other steps.

Make

  • New scenario → Webhooks → Custom webhook → paste the URL above.
  • Use “Determine data structure” after a test request, or the first live delivery.

n8n

  • Add a Webhook node (POST), activate, paste the production URL above.

Slack

Slack’s incoming webhooks expect their own JSON shape. Route this URL through Zapier, Make, or n8n first, then map our fields to Slack text or Block Kit.
